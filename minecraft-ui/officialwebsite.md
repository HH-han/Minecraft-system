# 设计规范：仿 Apple 官网视觉语言系统

## 1. 设计理念

Apple 官网的核心设计哲学可概括为：

- **极简主义**：去除一切不必要的装饰，内容即界面。
- **内容优先**：产品影像与文字为核心，背景留白充足。
- **清晰与层次**：通过字体大小、粗细、透明度建立信息层级。
- **流畅与细腻**：微交互、滚动视差、过渡动画增强沉浸感。
- **克制色彩**：以中性色（白、灰、黑）为主，蓝色为唯一强调色。

---

## 2. 色彩系统

| 用途 | 色值（HEX） | 备注 |
|------|-------------|------|
| 主文本 | `#1d1d1f` | 主要文字，高对比 |
| 次要文本 | `#6e6e73` | 说明性文字、价格、副标题 |
| 背景主色 | `#ffffff` | 纯白背景 |
| 背景辅色 | `#f5f5f7` | 卡片、次级区域背景 |
| 强调色（链接/按钮） | `#2997ff` | 悬停加深至 `#0066cc` |
| 分割线/边框 | `#d2d2d6` | 极浅灰 |
| 黑色背景区 | `#000000` | 用于高端产品（如 Watch） |
| 黑色背景上的文字 | `#ffffff` / `#a1a1a6` | 主/次 |

> **原则**：整体保持无彩色倾向，只保留蓝色作为交互反馈。

---

## 3. 字体系统

Apple 官网使用 **SF Pro** / **SF Pro Display** 作为英文字体，中文字体为 **苹方 (PingFang SC)**。本规范使用 `Inter` 作为降级替代。

| 层级 | 字号 | 字重 | 行高 | 使用场景 |
|------|------|------|------|----------|
| H1 超大标题 | 48px ~ 64px | 700 | 1.05 | Hero 产品主标题 |
| H2 大标题 | 32px ~ 44px | 700 | 1.1 | 区块主标题 |
| H3 卡片标题 | 24px ~ 28px | 600 | 1.2 | 产品卡片名称 |
| 正文 | 16px ~ 18px | 400 | 1.4 | 描述文字 |
| 辅助文字 | 12px ~ 14px | 400 | 1.4 | 脚注、价格、版权 |
| 导航链接 | 14px | 500 | 1 | 顶部导航栏 |

**响应式调整**：移动端标题缩小约 30%。

---

## 4. 间距与布局

- **网格基准**：8px 网格系统（所有间距为 8 的倍数）
- **内容区最大宽度**：1200px（居中，左右留白 22px）
- **卡片内边距**：24px ~ 40px
- **区块间距**：40px ~ 80px（垂直方向）
- **CTA 链接间距**：32px（两个链接之间）

**布局模式**：
- 单栏居中（Hero）
- 两列等宽卡片（产品并列）
- 三列等宽（配件/小产品）
- 全宽深色条（特色产品）

---

## 5. 组件规范

### 5.1 导航栏 (Navbar)

| 属性 | 值 |
|------|-----|
| 背景 | `rgba(255,255,255,0.72)` + `backdrop-filter: blur(20px)` |
| 高度 | 44px（实际内容区 12px 上下内边距） |
| 菜单项间距 | 32px |
| 图标区 | 右侧搜索 + 购物袋 |
| 移动端 | 菜单折叠为汉堡（或简化隐藏） |

### 5.2 按钮与链接

| 类型 | 样式 |
|------|------|
| 文本链接 | 颜色 `#2997ff`，无下划线，悬停下划线 |
| 次要 CTA | 同文本链接，后跟 `›` 图标 |
| 深色背景按钮 | 白底黑字，圆角 40px，内边距 12px 28px |

### 5.3 产品卡片

| 属性 | 值 |
|------|-----|
| 背景 | `#f5f5f7` 或纯白 + 细边框 |
| 圆角 | 24px ~ 28px |
| 内边距 | 40px 24px 30px |
| 悬停效果 | `scale(1.01)` + 轻微阴影 |
| 图片区域 | 居中，最大宽度 85% |

### 5.4 图片占位风格

- 使用**高分辨率产品渲染图**或实拍图
- 图片背景干净（透明或与卡片背景融合）
- 禁止使用杂乱的背景或边框

---

## 6. 交互动效

| 场景 | 动效参数 |
|------|----------|
| 链接/按钮悬停 | 颜色渐变，`transition: 0.2s ease` |
| 卡片悬停 | `transform: scale(1.01)`，`box-shadow` 出现 |
| 页面滚动 | 平滑滚动 `scroll-behavior: smooth` |
| 导航栏滚动 | 背景透明度微调（可选） |

> **原则**：动效应当微妙、快速、不干扰内容。

---

## 7. 响应式断点

| 断点 | 设备 | 布局调整 |
|------|------|----------|
| ≥ 1200px | 桌面 | 完整布局，两列/三列 |
| 768px – 1199px | 平板 | 字体缩小 20%，间距缩小 |
| ≤ 767px | 手机 | 所有卡片堆叠为单列，标题缩小，导航隐藏文字菜单 |

---

## 8. 页面区块结构（典型首页）

```
[导航栏]
[主视觉 Hero] — 产品名 + 副标题 + CTA + 大图
[双卡片区] — iPhone / MacBook 等
[全宽特色条] — Watch 或其他主力产品
[三列配件区] — AirPods / HomePod / 配件
[可选：更多产品]
[页脚 Footer]
```

---

## 9. 页脚规范

- 背景色：`#f5f5f7`
- 多列链接布局（移动端堆叠）
- 版权信息：`#6e6e73`，12px
- 包含：服务条款、隐私政策、销售政策等

## 10. 设计资源示例（代码片段）

以下为符合本规范的典型卡片 HTML/CSS 结构示例：

```html
<div class="product-card">
  <h3 class="card-title">iPhone 15 Pro</h3>
  <p class="card-desc">钛金属，强势出新。</p>
  <div class="cta-links">
    <a href="#">进一步了解 ›</a>
    <a href="#">购买 ›</a>
  </div>
  <div class="product-image">
    <img src="iphone.png" alt="">
  </div>
</div>
```

```css
.product-card {
  background: #f5f5f7;
  border-radius: 28px;
  padding: 40px 24px 30px;
  transition: transform 0.2s, box-shadow 0.2s;
}
.product-card:hover {
  transform: scale(1.01);
  box-shadow: 0 20px 30px -12px rgba(0,0,0,0.1);
}
```

## 11. 可访问性要点

- 颜色对比度符合 WCAG 2.1 AA 标准
- 所有交互元素有 `:focus` 状态
- 图片提供 `alt` 属性
- 字体使用相对单位（rem）
